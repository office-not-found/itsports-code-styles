# Nest.js  
==================

## Описание проекта  

Краткое описание задач и проблем, которые решает проект. Опишите, как проект помогает пользователям и какие ключевые функции предоставляет.

## Технологический стек  

Проект разработан с использованием следующих технологий:
- **Node.js** и **Nest.js** — фреймворк для создания масштабируемых серверных приложений.
- **PostgreSQL** — для хранения данных (возможно использование другой СУБД).
- **Swagger** — для автоматической генерации документации API.
- **Jest** — для написания и выполнения тестов.
- **ESLint** — для проверки кода на соответствие стилям и стандартам.

## Документация API  

Документация API, сгенерированная Swagger, доступна по [этой ссылке](http://localhost:3000/api-docs).

## Архитектура проекта  

Проект следует стандартной архитектуре Nest.js, которая описана в [официальной документации](https://docs.nestjs.com/). Основные модули и компоненты хранятся в директории `src`. Внешние вспомогательные функции, кастомные декораторы и другие переиспользуемые компоненты рекомендуется хранить в библиотеках, сгенерированных с помощью команды `nest g library my-library`. Это помогает поддерживать чистую и поддерживаемую архитектуру, а также облегчает повторное использование кода.

## Библиотеки

Для хранения переиспользуемого кода рекомендуется использовать библиотеки Nest.js. Подробнее о создании и использовании библиотек можно узнать в [документации Nest.js](https://docs.nestjs.com/cli/libraries).

## Стандарты кода (ESLint)  

Проект должен соответствовать критериям ESLint для поддержания единого стиля кодирования. Конфигурация ESLint находится на уровень выше в директории проекта.
